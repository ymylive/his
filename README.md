# 轻量级 HIS 课程设计

面向课程设计答辩演示的轻量级医院信息系统，覆盖挂号、接诊、检查、住院、病区、药房和管理员总览场景，提供桌面版可视化工作台。

## 演示账号

首次使用前，先记下以下账号。桌面版启动后可直接登录对应角色，登录页底部也会显示完整账号列表。

| 角色 | 账号 | 密码 | 说明 |
|------|------|------|------|
| 管理员 | `ADM0001` | `admin123` | 患者/科室/医生/病房/药品总览与维护 |
| 医生 | `DOC0001` | `doctor123` | 待诊列表、接诊录入、检查管理 |
| 住院管理员 | `INP0001` | `inpatient123` | 入院出院办理、病房床位调度、出院检查 |
| 药房 | `PHA0001` | `pharmacy123` | 药品建档、入库补货、发药处理 |
| 患者 | `PAT0001` | `patient123` | 自助挂号、看诊/住院/发药记录查询 |

## 演示数据说明

系统内置了一套完整的模拟数据，覆盖 30 名患者、10 名医生、6 个科室、5 个病区、12 种药品，以及挂号、看诊、检查、住院、发药等全流程记录。数据之间相互关联，可直接用于答辩演示。

如需恢复初始数据，可在登录页点击"重置演示数据"。

## 如何开始

建议直接使用 GitHub Release 中的预编译版本。当前推荐使用 `v2.4.0` 及以上版本：

- 最新版本：<https://github.com/ymylive/his/releases/latest>

### Windows

1. 打开最新 Release。
2. 下载 `lightweight-his-portable-v*-win*.zip`。
3. 解压到任意目录。
4. 双击 `run_desktop.bat` 启动桌面版。
5. 如果想恢复完整答辩环境，可在登录页点击"重置演示数据"。

### macOS

1. 打开最新 Release。
2. 下载 `lightweight-his-portable-v*-macos-arm64.zip`。
3. 解压后进入目录。
4. 首次运行先执行：

```bash
./setup-macos.sh
```

5. 再启动桌面版：

```bash
./run_desktop.sh
```

说明：

- macOS 版本当前面向 Apple Silicon（M1/M2/M3）预编译发布。
- 由于应用未经 Apple 签名，首次运行前必须执行 `setup-macos.sh`。
- 详细说明见 [macOS 安全配置指南](docs/MACOS_SECURITY.md) 和 [macOS 支持文档](docs/MACOS_SUPPORT.md)。
- 如需回到完整演示环境，可在登录页点击"重置演示数据"。

### Linux

当前仓库主要提供 Windows 和 macOS 的预编译桌面包。Linux 用户请按源码方式构建运行，见 [构建与运行指南](docs/BUILD_AND_RUN.md)。

## 推荐演示路线

如果你要做答辩演示，建议直接按下面 3 条路线体验，基本能覆盖系统的主要业务闭环。

1. 门诊闭环：患者登录自助挂号 -> 医生登录接诊 -> 药房登录发药 -> 患者重新登录查看发药记录和药品说明。
2. 检查闭环：患者挂号 -> 医生开立检查 -> 医生或相关角色回写结果 -> 患者查看检查摘要与病历信息。
3. 住院闭环：患者挂号 -> 医生建议住院 -> 住院管理员办理入院并分配床位 -> 转床调度和出院前检查 -> 办理出院。

## 功能总览

### 患者

- 自助挂号
- 查看看诊记录、检查摘要、住院状态
- 查看发药记录和药品说明
- 可直接搜索并选择科室、医生、药品，不必手输编号

### 医生

- 待诊列表
- 患者历史
- 接诊录入
- 检查管理与结果回写
- 可直接从待诊和患者列表中搜索后选择对象

### 住院管理员

- 入院登记与出院办理
- 住院查询
- 病房总览与床位状态
- 转床调度
- 出院前检查

### 药房

- 药品建档
- 入库补货
- 发药处理
- 库存查询与低库存预警

### 管理员

- 患者总览
- 科室与医生维护
- 病房与药品总览
- 系统信息查看

## 常见问题

### macOS 提示无法打开或已损坏

先在解压目录执行：

```bash
./setup-macos.sh
```

如果仍有问题，查看 [macOS 安全配置指南](docs/MACOS_SECURITY.md)。

### 中文显示异常或出现乱码

请优先使用最新版本。若仍异常：

- 确认你启动的是最新 Release
- 不要拆散 `data/` 目录和可执行文件
- 查看 [字体加载说明](docs/FONT_LOADING.md)

### 启动后数据不对或内容为空

请确认以下文件与程序放在同一解压目录中：

- `his_desktop` 或 `his_desktop.exe`
- `run_desktop.sh` 或 `run_desktop.bat`
- `data/`

不要只单独移动可执行文件。

如果你已经做过一轮演示、想恢复默认样例：

- 桌面版：登录页或管理员系统页点击"重置演示数据"
- 控制台版：主菜单选择"重置演示数据"
