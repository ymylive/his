# README 用户手册化改造设计

## 目标

将仓库根目录 `README.md` 从“用户说明 + 开发说明混合页”改为“最终使用者优先”的使用手册，让第一次打开仓库或 Release 的用户可以直接完成以下任务：

- 找到全部演示账号
- 下载正确版本
- 在 Windows / macOS / Linux 上启动程序
- 按推荐顺序体验完整业务演示
- 遇到常见问题时快速定位解决方法

开发者构建、测试、打包信息不再占据 README 主体，只保留简短入口，详细内容继续留在 `docs/`。

## 目标读者

主要读者：

- 课程答辩演示者
- 首次下载 Release 的普通用户
- 需要快速试用系统功能的评审或老师

次要读者：

- 想从源码构建的开发者

README 不再以开发者为主线组织内容。

## 信息架构

README 采用以下顺序：

1. 项目简介
2. 演示账号
3. 如何开始
4. 推荐演示路线
5. 功能总览
6. 常见问题
7. 更多文档

## 各部分设计

### 1. 项目简介

保留一句简洁说明：

- 这是一个面向课程设计答辩的轻量级 HIS
- 技术栈和跨平台能力可简述，但不展开构建细节

目的：

- 用户先知道这是什么
- 不在开头塞过多实现细节

### 2. 演示账号

将全部 7 个演示账号前置到 README 顶部可见区域。

账号列表统一完整呈现：

- `ADM0001 / admin123`
- `CLK0001 / clerk123`
- `DOC0001 / doctor123`
- `INP0001 / inpatient123`
- `WRD0001 / ward123`
- `PHA0001 / pharmacy123`
- `PAT0001 / patient123`

设计要求：

- 不再只展示 3 个账号
- 每个账号标注角色中文名称
- 放在“如何开始”之前，避免用户启动后才发现不知道怎么登录

### 3. 如何开始

按最终使用者视角组织，不按开发者视角组织。

包含：

- Windows 用户：下载最新 Release、解压、运行 `run_desktop.bat`
- macOS 用户：下载 `lightweight-his-portable-v*-macos-arm64.zip`、解压、运行 `./setup-macos.sh`、再运行 `./run_desktop.sh`
- Linux 用户：明确说明当前以源码运行或本地构建为主

额外要求：

- macOS 安全配置警告保留，但压缩到用户必要步骤
- 明确推荐下载 `v2.0.5` 及以上版本，避免用户误用旧版错误资产

### 4. 推荐演示路线

新增一个“从零到完整业务闭环”的路线说明，用于答辩或演示。

建议写成 3 条主线：

1. 门诊挂号到接诊到发药
2. 挂号到检查到结果回写
3. 挂号到住院到病区到出院

并补充每条主线建议使用的角色：

- 患者
- 挂号员
- 医生
- 住院登记员
- 病区管理员
- 药房
- 管理员

目的：

- 用户不只是“能登录”
- 用户知道“下一步该点什么”

### 5. 功能总览

保留角色工作台总览，但压缩表述，减少营销式重复描述。

每个角色保留一句：

- 患者：挂号、看诊、住院、发药、药品说明
- 挂号员：建档、查询、挂号、取消
- 医生：待诊、历史、接诊、检查
- 住院登记员：入院、出院、住院查询
- 病区管理员：病房、床位、转床、出院检查
- 药房：建档、入库、发药、库存、预警
- 管理员：患者总览、科室医生、病房药品、系统信息

### 6. 常见问题

README 中只保留最终使用者最常遇到的 3 类问题：

- macOS 首次无法打开
- 中文显示异常
- 数据目录或启动脚本位置错误

每类问题只给最短处理方式，并链接到详细文档。

### 7. 更多文档

文档入口统一收敛到 README 末尾：

- 安装依赖
- 构建与运行
- 字体加载说明
- macOS 支持
- macOS 安全配置
- 更新日志

其中开发者最常用的“构建与打包”不再放在 README 主流程中。

## 明确删减内容

以下内容从 README 主体移除或下沉：

- 大段源码构建命令
- 打包脚本说明作为主流程内容
- Docker 作为主要使用路径
- 重复出现的“快速开始”与“演示账号”
- 过长的 UI 强化亮点描述

这些内容仍可保留在 `docs/BUILD_AND_RUN.md`、`docs/INSTALL.md` 等文件中。

## 文案风格

README 应满足：

- 以“能用”为第一优先级
- 使用短段落和扁平列表
- 尽量减少重复宣传性形容词
- 先给操作，再给解释
- 明确区分“预编译使用”和“源码构建”

## 约束

- 不修改已有详细文档主体内容
- 只重构 `README.md`
- 所有 Release 资产名以当前真实发布产物为准
- 不在 README 中继续引用已经清理掉的错误 macOS 资产名

## 成功标准

改造完成后，用户仅阅读 `README.md` 即可完成：

- 找到可用演示账号
- 下载正确安装包
- 成功启动桌面版
- 知道如何走完整演示流程
- 出现常见问题时知道去哪里处理
